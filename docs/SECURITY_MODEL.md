# Security Model

IGRIS_GPT adopts a defence‑in‑depth approach to security.  Running an autonomous agent that can read and act on code repositories requires careful constraints to avoid data leakage or unintended actions.

## Principles

1. **Least privilege** – The agent operates with only the permissions it needs.  It cannot access arbitrary paths or execute arbitrary commands.
2. **Explicit allowlists** – Sensitive operations such as running commands, making network requests or writing files are gated behind allowlists.  Disallowed actions are refused.
3. **Transparent execution** – All actions (messages sent to the LLM, commands executed, files read) are logged and available via the logs API.
4. **No secrets in code** – API keys and other sensitive data are loaded from environment variables.  Example configuration files are provided but secrets are never committed to the repository.

## Safety Shield

The safety shield monitors tasks and plans generated by the agent.  It can block tasks that repeatedly belong to the same family (anti‑loop), tasks that attempt to exfiltrate code, or tasks that appear malicious.  The `igris/core/anti_loop.py` module contains simple heuristics for family classification and saturation thresholds.

## Risk Categories

* **Command execution** – mitigated via the safe terminal and test runner.
* **File access** – restricted to the project root; the API rejects path traversal patterns such as `..`.
* **Network calls** – disabled by default.  Future versions may allow external API calls with configurable allowlists.
* **LLM hallucinations** – addressed by combining multiple models and including a teacher/advisor loop to cross‑check outputs.
